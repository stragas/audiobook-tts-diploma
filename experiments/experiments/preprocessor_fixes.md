# Исправление критического бага: точки внутри тегов пауз

## Проблема
Регекс `([.!?])\s*` для расстановки пауз в конце предложения 
матчил точку внутри тега `[spk=1.5]`, превращая его в:
`[spk=1.[spk=1.0] 5]`

## Причина
Регулярное выражение не учитывало контекст — точка внутри 
квадратных скобок не должна обрабатываться как конец предложения.

## Решение
Изменены регексы для проверки контекста:

### Было (неправильно):
```python
text = re.sub(r'([.!?])\s*', f'\\1 {self.PAUSE_LONG} ', text)

### Стало (правильно):
```python
# Точка должна сопровождаться пробелом или концом строки
text = re.sub(r'([.!?])(\s|$)', f'\\1{self.PAUSE_LONG}\\2', text)

Проверка корректности
Добавлены автоматические тесты:
Проверка целостности тегов (все заканчиваются на ])
Проверка на вложенность тегов (нет [spk=...[spk=...)
Подсчёт корректных тегов через валидный регекс
